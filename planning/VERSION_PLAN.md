## 📊 版本管理规范

### 分支策略
```
main        ────●───────────●───────────●───>  # 稳定发布
              v0.1        v0.2        v1.0

develop     ────┼───┬───────┼───┬───────┼───>  # 开发主线
                │   │       │   │       │
feature/v0.1  ───┘   │       │   │       │
feature/v0.2  ───────┘       │   │       │
feature/v0.3  ───────────────┘   │       │
...                             │       │
feature/v1.0  ────────────────────────────┘
```

### 提交规范
1. **每个版本完成**后创建版本分支：`git checkout -b release/vX.Y`
2. **版本测试通过**后合并到develop：`git checkout develop && git merge release/vX.Y`
3. **版本发布**时：
   - 合并到main分支：`git checkout main && git merge develop`
   - 创建标签：`git tag -a v.X.Y -m "Release version X.Y"`
   - 推送到远程：`git push origin v.X.Y`

### 任务管理
- 使用`TASK-XXX`格式标记任务ID
- 每个任务必须关联到具体版本
- 完成任务前必须通过代码审查
- 任务状态在`planning/tasks_status.md`中跟踪
