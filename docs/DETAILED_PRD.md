## 📄 AI 模拟对话应用 - 详细需求文档

### 1. 产品概述

#### 1.1 产品愿景
**打造行业领先的AI对话模拟平台**，通过高度拟真的多角色对话场景，帮助用户在安全环境中提升沟通能力、面试技巧和社交自信。  
**核心理念**：*"Practice makes perfect"* —— 让每一次对话练习都带来真实进步。

#### 1.2 产品定位
| 维度 | 描述 |
|------|------|
| **目标用户** | 求职者、职场新人、社交焦虑者、企业培训部门 |
| **核心价值** | 提供可定制的AI对话场景 + 智能反馈 + 数据化成长追踪 |
| **差异化优势** | **场景深度**（5+专业场景）<br>**角色真实度**（20+角色属性）<br>**反馈精准度**（7维度量化评估） |
| **商业模式** | 免费基础版 + 企业定制版（SaaS订阅） |

#### 1.3 核心场景矩阵
| 场景类型 | 子场景 | 关键痛点 | 解决方案 |
|----------|--------|----------|----------|
| **面试场景** | 技术面试<br>HR面试<br>压力面试 | 紧张、回答不专业 | 模拟真实面试官风格<br>提供结构化回答建议 |
| **沙龙场景** | 技术沙龙<br>社交聚会<br>学术讨论 | 话题中断、表达混乱 | 多AI角色主动引导话题<br>实时话术提示 |
| **项目会议** | 需求评审<br>进度同步<br>冲突解决 | 逻辑不清、抓不住重点 | 角色分工明确<br>会议结构化引导 |
| **商务谈判** | 价格谈判<br>合作洽谈 | 缺乏谈判策略 | 提供谈判策略库<br>实时胜率预测 |

### 2. 用户需求深度分析

#### 2.1 典型用户画像
**求职者小王（核心用户）**  
- **背景**：2年经验的前端工程师，准备跳槽  
- **痛点**：  
  - 面试时经常被问倒，尤其是系统设计题  
  - 不知道如何组织语言回答行为面试题  
  - 无法判断自己的回答是否达标  
- **期望**：  
  - 获得技术面试官的真实反馈  
  - 学习高分回答的思维框架  
  - 模拟压力面试环境  

#### 2.2 用户旅程地图
```mermaid
journey
    title 用户使用流程
    section 准备阶段
      选择场景 --> 配置参数 --> 查看角色介绍
    section 对话阶段
      开始对话 --> AI提问 --> 用户回答 --> 实时提示
    section 反馈阶段
      生成报告 --> 查看弱点 --> 获取练习建议 --> 保存记录
```

#### 2.3 需求优先级排序（MoSCoW法则）
| 需求 | 优先级 | 说明 |
|------|--------|------|
| 支持面试/沙龙/会议3种核心场景 | **MUST** | MVP必备 |
| AI角色具备差异化性格特征 | **MUST** | 提升真实感 |
| 对话后生成7维度评分报告 | **MUST** | 核心价值点 |
| 用户自定义角色属性 | **SHOULD** | 增强个性化 |
| 语音对话支持 | **COULD** | 未来扩展 |
| 实时情绪识别调整对话 | **WON'T** | 技术难度高 |

### 3. 功能规格说明书

#### 3.1 场景配置系统
**面试场景示例配置**：
```yaml
scene: interview
type: technical
parameters:
  interviewer_style: "pressure"  # 可选：friendly/strict/pressure
  domain: "system_design"       # 技术领域
  duration: 30                  # 分钟
  difficulty: 3                 # 1-5级
roles:
  - name: "张总监"
    position: "技术总监"
    traits:
      communication: "direct"
      evaluation: "strict"
      patience: 3/5
      expertise: 5/5
  - name: "李经理"
    position: "研发经理"
    traits:
      communication: "casual"
      evaluation: "balanced"
      patience: 4/5
      expertise: 4/5
```

#### 3.2 角色行为规范
**技术面试官行为规则**：
| 行为类型 | 规则描述 | 触发条件 |
|----------|----------|----------|
| 提问策略 | 从基础到深入递进 | 初次交互 |
| 追问机制 | 针对模糊回答追问细节 | 回答不完整时 |
| 压力测试 | 故意打断/质疑 | 检测到用户紧张 |
| 反馈方式 | 先肯定后建议 | 每轮对话结束 |

#### 3.3 智能反馈系统
**7维度评分模型**：
```python
def evaluate_dialogue(dialogue_history):
    return {
        "content_quality": {
            "score": 4.2,
            "criteria": [
                "技术准确性 (4.5)",
                "逻辑完整性 (3.8)",
                "案例丰富度 (4.0)"
            ]
        },
        "expression_clarity": {
            "score": 3.9,
            "criteria": [
                "语言简洁度 (3.5)",
                "重点突出 (4.2)",
                "避免口头禅 (4.0)"
            ]
        },
        # ... 其他5个维度
    }
```

**反馈报告示例**：
```
【系统设计面试反馈】
✅ 优势：  
- 清晰表达了微服务拆分原则（4.7/5）  
- 使用了电商案例说明（4.5/5）  

⚠️ 待改进：  
- 未提及服务治理方案（2.8/5）  
  → 建议学习：服务注册发现、熔断机制  
- 回答时间过长（3分15秒）  
  → 练习：用STAR法则结构化回答  

📈 成长建议：  
1. 重点练习：服务治理知识  
2. 推荐资源：《微服务架构设计模式》第5章  
3. 专项训练：30秒快速回答练习
```

**产品经理备注**：  
> 本产品不是简单的"AI聊天机器人"，而是**结构化的对话训练系统**。关键在于：  
> 1️⃣ **场景真实性** - 模拟真实职场对话的复杂性  
> 2️⃣ **反馈专业性** - 提供可操作的改进建议  
> 3️⃣ **成长可视化** - 用数据见证能力提升  
>  
> 首期将聚焦技术面试场景打磨核心体验，后续扩展至更广的对话训练领域。  
> **记住：用户买的不是功能，而是更好的自己。**
